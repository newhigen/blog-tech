## 문제의식

근태 시스템에서 export 받은 데이터는 일 단위 raw에 불과하다. 다음 질문에 답하지 못한다.

- 야근이 **어느 시기에** 몰렸는가
- 출장·휴가 이후 출근 리듬이 어떻게 변했는가
- 분기·연도별 **변곡점**이 언제인가
- 출퇴근 시간의 분포가 어떻게 이동했는가

엑셀 한 장으로는 한눈에 안 보인다. 시계열·분포·요약을 한 화면에 모아야 패턴이 드러난다.

## 접근

단일 HTML + Chart.js로 구성한다. 외부 서버 없이 브라우저에서 직접 실행되며 데이터는 파일에 embed.

### 빌드 파이프라인

```mermaid
flowchart LR
  A[근태 export xlsx] -->|analyze.py| B[days.json]
  B -->|build.py| C[통계·KPI·변곡점]
  C --> D[index.html]
  D --> E[Dashboard]
```

`build.py` 한 번 실행으로 866일 raw → 13개 차트 + 5개 KPI + 변곡점 annotation 자동 생성.

### 도메인 규칙

| 항목 | 규칙 |
|------|------|
| 실근무 시간 | (퇴근 − 출근) − 점심 90분 − 저녁 60분(퇴근≥20:00) |
| 야근 등급 | 실근무 9h+ 유연 / 10h+ 일반 / 11h+ 장야근 |
| 변곡점 감지 | 전월 대비 출근시간 ±30분 이상 변화 |

야근 기준은 **퇴근 시각이 아니라 실근무 시간**이다. 늦게 출근해서 늦게 퇴근하면 야근 아니다.

## 시각화 구성

- **연간 히트맵** — GitHub 스타일 그리드, 근무 유형별 색상 + 야근 강도 테두리 링
- **출퇴근 분포** — 15분 버킷 히스토그램
- **월별 평균 라인** — Y축 reverse(위=아침), 연도별 오버레이
- **변곡점 annotation** — 30분 이상 출근 시간 변화 시점 자동 마킹
- **월별 변동성** — min/max 범위 + IQR 박스 + 평균선

## 디자인 결정

- Toss 다크모드 디자인 시스템 참조 — flat card, KPI 숫자 위계
- 색상은 **빈도 → 채도**로 매핑. 자주 보이는 패턴이 기본 색이어야 맥락이 즉시 읽힌다
- 야근 표시는 셀 색이 아닌 **테두리 링** (amber→orange, 강도별 굵기)으로 분리 — 근무 자체와 강도 정보가 겹치지 않게

## 작업 과정

1. 근태 export 파싱 규칙 정의 (analyze.py)
2. 차트 13종 설계 — 시계열·분포·변곡점 카테고리 분리
3. `build.py` 작성 — days.json 입력 → 모든 통계 자동 계산
4. dummy data generator 추가 — 실데이터 없이도 빌드 가능
5. 디자인 시스템 정리 (`DESIGN.md`) — 색상 토큰, 카드 패턴, 차트 컨벤션
