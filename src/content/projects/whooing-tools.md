## 문제 정의

복식부기 가계부는 입력 규칙이 엄격하다. 은행 거래내역을 그대로 붙여넣을 수 없고, 차변·대변 계정을 직접 지정해야 한다. 매달 비슷한 작업을 반복하는데 기준이 흔들리면 분류가 뒤섞인다.

특히 여러 은행·카드사 명세서의 형식이 다 달라서 수작업 변환에 매달 45~55분이 소요됐다.

## 과정

1. **Before 측정** — 기존 워크플로우 기록. 월별 수작업 시간, 반복 패턴, 오류 발생 지점 파악.
2. **자동화 범위 정의** — 4가지 자동화 아이디어(명세서 변환, HITL 확인, 구독 모아보기, 지출 분석)를 우선순위별로 정리.
3. **파서 개발** — `parse_bank.py`로 은행별 Excel/CSV 형식을 읽어 후잉 import 포맷으로 변환. 계정 분류 규칙은 별도 문서화.
4. **워크플로 문서화** — 계정 구조, 월간 정산 규칙, 분류 기준을 wiki에 정리해 일관성 유지.

## 주요 기능

- **명세서 변환** — 은행 거래내역을 후잉 import CSV로 자동 변환
- **계정 분류** — 거래 패턴 기반 차변·대변 자동 매핑
- **워크플로 문서** — 계정 구조와 월간 정산 규칙 정리
- **리뷰 기록** — 월별 거래 검토 메모 누적

## 구현 포인트

- Python 스크립트로 반복 수작업 제거
- 데이터 정제 규칙과 회계 규칙을 함께 문서화
- 단순 스크립트보다 **운영 가능한 개인 시스템**에 가깝게 구성
