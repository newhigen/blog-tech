## 문제 정의

보험은 문서가 길고 용어가 복잡하다. 실손과 정액형이 섞이면 "무엇이 중복되고 무엇이 아닌지"가 흐려진다. 여러 상품을 같은 기준으로 비교하려면 제안서를 직접 펼쳐 대조해야 하는데, 이 과정에서 핵심을 놓치기 쉽다.

특히 한국 보험 구조(실손 세대별 차이, 갱신형 함정, 정액 중복수령)를 모르면 비교 자체가 어렵다.

## 과정

1. **구조 파악** — 한국 보험의 핵심 개념(실손 1~4세대, 갱신형 vs 비갱신형, 정액 중복수령)을 정리해 비교 기준 수립.
2. **데이터 수집** — 현재 가입 보험과 제안받은 보험 상품의 보험료, 주요 담보를 PDF에서 추출.
3. **비교 프레임 설계** — 공통 베이스 조건 + A안 vs B안 구조로 항목 정리. 시너지 분석(중복수령 가능 조합) 별도 섹션 구성.
4. **대시보드 빌드** — 정적 HTML로 브라우저에서 바로 실행. `insurance_dashboard.html`, `comparison_report.html`, `korea_insurance_guide.html` 세 파일로 구성.

## 주요 기능

- **상품 비교** — 보험료와 핵심 담보를 같은 축에서 비교
- **실손 구조 요약** — 세대별 차이와 갱신형 특성 정리
- **정액 보장 조합** — 중복 수령 가능성 확인
- **시너지 분석** — A+B 조합 시 중복 수령 가능 항목 명시
- **의사결정 보조** — A안/B안과 공통 조건을 분리해 정리

## 구현 포인트

- 브라우저에서 바로 여는 정적 HTML 대시보드
- 한국 보험 문맥에 맞는 설명과 표 구성
- 단순 시각화보다 실제 비교 판단에 필요한 텍스트 구조 강조
